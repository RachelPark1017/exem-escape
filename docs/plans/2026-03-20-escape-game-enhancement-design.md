# EXEM 방탈출 게임 고도화 디자인

## 개요
EXEM 사무실 탈출 게임의 UI/UX 개선, 애니메이션 강화, 힌트 시스템 추가, 퍼즐 다양화, 스토리 강화를 위한 디자인 문서.

**접근 방식**: 단일 index.html 구조를 유지하며 점진적 개선 (A안)
**사용 환경**: 모바일 + PC 양방향 최적화

---

## 1. UI/UX 디자인 리뉴얼

### 인트로 화면
- 풀스크린 시네마틱 인트로
- 어두운 사무실 배경 위 타이핑 애니메이션으로 스토리 한 줄씩 등장
- 로고 글로우 효과, "탈출 시작하기" 버튼 펄스 애니메이션
- 스테이지 뱃지를 미니맵 스타일로 변경 (아이콘만 보이다가 호버 시 이름 표시)

### 게임 화면
- 카드 UI: 글래스모피즘(반투명 유리) 효과, 깊은 그림자
- 프로그레스 바: 상단에 전체 진행도 (5단계) + 스테이지 내 진행도 분리
- 타이머: 원형 프로그레스 바, 시간 경과에 따라 색상 그라데이션 변화

### 엔딩 화면
- 등급 시스템 (S/A/B/C) 표시
- 스테이지별 정답률 그래프
- 공유 가능한 결과 카드

### 1인칭 시점 연출
- CSS 3D perspective 활용 "걸어가는" 전환 애니메이션 (줌인 → 페이드 → 줌아웃)
- 방 진입 시 "문을 열고 들어가는" 시점 변화
- 패럴랙스 배경 반응 (마우스 이동/디바이스 기울기)
- 퀴즈 오브젝트 "발견하는" 느낌 애니메이션

---

## 2. 애니메이션 & 인터랙션

### 스테이지 전환
- 3D 공간 이동: `perspective(800px) translateZ()` 활용
- 문 열림 애니메이션: 검은 화면이 가운데에서 양옆으로 갈라짐
- 모바일 미세 진동: `navigator.vibrate()` (지원 시)

### 퀴즈 인터랙션
- 정답: 초록 플래시 + 잠금장치 풀리는 CSS 애니메이션
- 오답: 붉은 깜빡임 + 진동 강화
- 빈칸 입력: 타자기 커서 깜빡이 효과
- 선택지: 호버 시 확대 + 글로우

### 스토리 연출
- 방 진입 시 내레이션 타이핑 속도 조절 (긴장감 부분 느리게)
- 힌트 발견 시 "발견!" 팝업 + 반짝이 파티클
- 타이머 3분 이하 시 긴장감 스토리 삽입

---

## 3. 힌트 시스템

### 구조
- 문제당 3단계 힌트:
  - 1단계: 방향 힌트 (감점 없음)
  - 2단계: 구체적 힌트 (50점 감점)
  - 3단계: 거의 정답 (100점 감점)

### UI
- 하단 전구 아이콘 버튼 (💡) 상시 표시
- 클릭 시 힌트 카드 슬라이드업
- 남은 힌트 횟수 숫자 뱃지
- 점수 차감 애니메이션

### 엔딩 반영
- 결과에 힌트 사용 횟수 표시
- 힌트 0번 → "독립 탈출" 특별 뱃지

## 4. 콘텐츠: 새로운 퍼즐 유형

기존: OX, 객관식(choice), 빈칸(fill), 시나리오(scenario)

### 추가 퍼즐 유형
1. **드래그 & 드롭 매칭 (match)** — 항목 연결 (예: 핵심가치 ↔ 정의)
2. **순서 배열 (order)** — 카드를 올바른 순서로 정렬
3. **이미지 선택 (image)** — 상황 사진/아이콘 중 선택
4. **비밀번호 다이얼 (dial)** — 금고 다이얼 UI로 숫자/글자 입력

## 5. 콘텐츠: 스토리 강화

### 컷씬
- 스테이지 사이 1인칭 내레이션 이동 과정 묘사
  - 예: "서랍에서 찾은 카드를 손에 쥐고 파일 캐비닛으로 향합니다. 복도의 형광등이 깜빡입니다..."

### 긴장감 요소
- 타이머 3분 이하: "형광등이 하나씩 꺼지기 시작합니다..."
- 배경 어두워지는 효과 연동

### 엔딩 에필로그
- 탈출 후 다음 날 아침 출근 장면으로 마무리

### 단서 요약
- 스테이지 클리어 시 발견한 단서 시각적 요약
